## 1. 产品概述

### 1.1 背景与目标

随着即时通讯的普及，人们每天需要处理大量聊天消息。许多人面临"不知道怎么回"、"回复效率低"、"沟通技巧欠缺"等痛点。本产品旨在通过AI技术，为用户提供智能回复建议，帮助用户在不同场景下都能进行有效沟通。

### 1.2 核心价值主张

| 价值点 | 说明 |
|--------|------|
| **智能回复** | 一键生成符合人设的回复内容 |
| **性格侧写** | 记住对话历史，通过AI分析形成性格侧写，动态更新人设标签权重 |
| **人设定制** | 用户自定义人设卡片（关系+标签+权重），生成不同风格的回复 |
| **意图洞察** | 分析对方意图，提供实时反馈（如价格提醒、时间提醒） |
| **智能总结** | 自动总结对话内容，提取关键信息 |

### 1.3 目标用户

| 用户类型 | 使用场景 | 核心需求 |
|----------|----------|----------|
| 日常用户 | 微信、QQ等日常聊天 | 趣味、快速、有温度的回复 |
| 职场人士 | 工作沟通、邮件、钉钉等 | 专业、清晰、高效的回复 |
| 销售/微商 | 客户咨询、转化跟进 | 专业顾问、促单、客情维护 |
| 客服人员 | 客户咨询、问题处理 | 专业、耐心、标准化回复 |

### 2.2 Local-First 架构

本产品采用 **Local-First**（本地优先）架构设计：

| 特性 | 说明 |
|------|------|
| **数据存储** | 所有数据优先本地存储，确保离线可用 |
| **标签初始权重** | 人设标签默认权重为 0.5（0.1-0.9范围，保留极端值空间） |
| **AI智能调整** | 用户开启后，自动同步聊天记录到服务端分析 |
| **隐私优先** | 默认关闭 AI 智能调整，不主动上传数据 |

**AI智能调整流程**：
```
客户端（开启AI智能调整）
    ↓ 同步聊天记录（仅对话内容，不含隐私信息）
服务端
    ↓ 定时分析（每日/每周）
    ↓ 计算标签权重调整建议
    ↓ 同步回客户端
客户端（自动更新标签权重）
```

> **注意**：上下文批次（用于生成回复的对话历史）与 AI 智能调整是两个独立流程：
> - 上下文批次：本地存储，用于生成回复时提供上下文
> - AI 智能调整：需上传服务端分析，用于自动调整标签权重

#### 3.2.5 AI 回复生成

| 功能 | 说明 |
|------|------|
| 基础生成 | 基于对方消息 + 场景 + 人设 + 关系生成回复 |
| 上下文感知 | 结合历史对话（对方的话 + 生成的回复）生成更精准的内容 |
| 多条建议 | 一次生成3条不同风格的回复供选择 |
| 复制/编辑 | 用户可直接复制或编辑后再发送 |

**生成逻辑**：
```
AI输入 = 对方消息 + 当前场景 + 人设标签(带级别) + 关系 + 历史上下文
    ↓
AI处理 = 意图识别 + 风格匹配 + 上下文整合
    ↓
AI输出 = 3条回复建议（不同表达方式）
```

#### 3.2.6 上下文学习引擎

> **重要澄清**：本功能分为两个独立模块

##### 模块A：上下文批次（本地存储，用于生成回复）

| 功能 | 说明 |
|------|------|
| 上下文批次 | 用户对话历史，用于生成回复时提供上下文理解 |
| 自动保存 | 每次生成回复后自动保存"对方消息+生成回复"到当前批次 |
| 本地存储 | 所有数据保存在本地，不上传服务端 |

**批次划分规则**：
- 同一时间段（默认2小时内）的连续对话为"一轮"
- 同一轮次内的话题具有高相关性
- 跨轮次可追溯历史形成性格侧写

##### 模块B：AI智能调整（需开启开关）

> 基于 Local-First 架构，需用户主动开启"AI智能调整"开关

| 功能 | 说明 |
|------|------|
| AI智能调整开关 | 默认关闭，开启后触发云端分析流程 |
| 标签初始权重 | 新建人设卡片时，所有标签权重默认为 0.5 |
| 云端分析 | 开启后自动同步聊天记录到服务端分析 |
| 动态权重更新 | 服务端计算权重调整建议，同步回客户端更新 |
| 手动覆盖 | 用户可随时手动修改标签权重，覆盖AI自动更新 |

**AI智能调整流程**：
```
用户开启"AI智能调整"开关
    ↓
客户端自动同步聊天记录到服务端（加密传输）
    ↓
服务端每日定时分析对话批次
    ↓
计算标签权重调整建议
    ↓
同步回客户端 → 自动更新标签权重
    ↓
用户可手动调整或关闭开关
```

**用户控制**：
- 用户可随时在卡片设置中手动调整任意标签权重
- 手动修改会标记为"用户自定义"，不会被AI自动更新覆盖
- 提供"重置为AI建议"按钮恢复自动调整
- 可随时关闭"AI智能调整"开关停止同步

#### 3.2.7 意图分析

| 意图类型 | 识别示例 | 反馈机制 |
|----------|----------|----------|
| 价格意图 | "这个多少钱"、"便宜点"、"预算500" | 提醒用户设置价格预算 |
| 时间意图 | "什么时候截止"、"几点到期"、"后天" | 标记时间紧迫性 |
| 确认意图 | "真的假的"、"确定吗"、"再确认下" | 反馈对方急切程度 |
| 情绪意图 | 识别对方情绪（积极/消极/急切/犹豫） | 提供沟通建议 |
| 承诺意图 | "我下次请你吃饭"、"周一给你" | 跟踪待办事项 |

**实时反馈示例**：
```
对方：便宜点呗，500以内可以考虑

```
