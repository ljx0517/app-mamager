# 聊商 - 交互优化方案

> 本文档记录交互优化方向，核心方案已确认
> **版本**：V1.1 | **更新日期**：2026-02-16

---

## 一、核心交互优化

### 1.1 交互流程对比

| 版本 | 步骤 | 操作数 |
|------|------|--------|
| **优化前** | 复制→呼出键盘→选人设→选场景→生成→选一条→发送 | 7步 |
| **优化后** | 复制→点击人设按钮→AI生成→自动填入→发送 | 4步 |

### 1.2 优化要点

| 优化点 | 说明 |
|--------|------|
| **人设按钮化** | 卡片 → 按钮，一目了然 |
| **记住上次选择** | 键盘记住上次使用的人设，下次自动选中 |
| **自动填入** | AI生成后自动填入聊天输入框 |
| **直接发送** | 无需切换APP，直接发送 |
| **场景内嵌** | 场景融入卡片名称，如 to老板【工作】 |

## 三、人设卡片命名规则

### 3.1 命名格式

```
人设卡片 = 联系人 + 【场景标签】 + 可选描述

示例：
├── to老板【工作】      → 谈工作、汇报、请假
├── to老板【日常】      → 闲聊、逢年过节问候
├── to女朋友【恋爱】    → 甜言蜜语、哄人
├── to女朋友【吵架】    → 道歉、认错、求和
├── to女朋友【日常】    → 日常关心、闲聊
├── 闺蜜【八卦】        → 吐槽、分享、聊八卦
├── to同事【协作】      → 工作配合、进度同步
├── to客户A【销售】     → 推销、跟进
├── to客户A【维护】     → 客情维护、节日问候
└── to相亲对象【初识】  → 互相了解、破冰
```

### 3.2 场景说明

| 场景 | 适用场景 | 典型人设标签 |
|------|----------|--------------|
| 【工作】 | 职场沟通、工作汇报 | 干练、务实、成熟、简洁 |
| 【日常】 | 非工作时间闲聊 | 幽默、温柔、被动 |
| 【恋爱】 | 追求期、甜蜜期 | 浪漫、温柔、调皮 |
| 【吵架】 | 矛盾、道歉、求和 | 真诚、温柔、成熟 |
| 【销售】 | 推销、跟进、转化 | 热情、专业、主动 |
| 【维护】 | 客情维护、节日问候 | 贴心、务实、专业 |
| 【八卦】 | 闺蜜/死党闲聊 | 幽默、调皮、八卦 |
| 【初识】 | 相亲、认识新朋友 | 真诚、礼貌、主动 |

## 七、优化对比

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| **操作步骤** | 7步 | 4步 |
| **人设选择** | 卡片形式，需点击展开 | 按钮形式，一目了然 |
| **场景设置** | 单独选择场景 | 融入卡片名称 |
| **回复填入** | 复制/手动粘贴 | **自动填入输入框** |
| **发送流程** | 复制→切换APP→粘贴→发送 | **直接发送** |
| **效率** | 30秒+ | **10秒内** |
