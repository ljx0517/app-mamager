> 本文档定义产品上线后的数据采集方案，支持数据驱动决策和持续优化
> **版本**：V1.0 | **更新日期**：2026-02-16

## 1. 指标体系

### 1.1 北极星指标（North Star Metric）

| 指标 | 定义 | 目标值（MVP） |
|------|------|----------------|
| **日活跃用户（DAU）** | 每日使用 聊商 生成回复的用户数 | 首月目标：5万 |
| **次日留存率** | 次日回访用户 / 当日新增用户 | > 30% |
| **人均生成回复数** | 每日AI生成回复总数 / DAU | > 10次/天 |
| **人设卡片使用率** | 使用人设卡片生成回复的用户占比 | > 50% |

### 1.2 健康指标

| 指标类别 | 指标名称 | 定义 | 预警阈值 |
|----------|----------|------|----------|
| **用户获取** | 新用户注册数 | 首次启动并完成引导的用户 | - |
| | 激活率 | 完成首次AI生成的用户/新增用户 | < 40% |
| **用户留存** | 次日留存 | T+1 回访率 | < 20% |
| | 7日留存 | T+7 回访率 | < 10% |
| | 30日留存 | T+30 回访率 | < 5% |
| **用户活跃** | 周活跃天数 | 用户每周使用天数 | < 2天 |
| | 核心功能使用率 | 使用人设/场景功能的用户占比 | < 30% |
| **商业变现** | 付费转化率 | 付费用户/活跃用户 | < 1% |
| | ARPU | 每用户平均收入 | - |

### 1.3 性能指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| 键盘呼出成功率 | 成功呼出次数/呼出请求次数 | > 99% |
| AI生成成功率 | 成功生成次数/生成请求次数 | > 95% |
| 键盘呼出耗时 | 从点击到键盘完全展示 | < 1秒 |
| AI生成耗时 | 从请求到返回结果 | < 3秒 |
| 崩溃率 | 崩溃次数/启动次数 | < 0.5% |
| ANR 率 | 应用无响应次数/启动次数 | < 0.1% |

## 2. 事件体系

### 2.1 事件分类

```
事件体系
├── 用户事件（User Events）
│   ├── 启动事件
│   ├── 登录事件
│   └── 设置事件
│
├── 核心功能事件（Core Feature Events）
│   ├── 键盘事件
│   ├── 人设事件
│   ├── AI生成事件
│   └── 上下文学习事件
│
├── 业务事件（Business Events）
│   ├── 付费事件
│   ├── 分享事件
│   └── 反馈事件
│
└── 错误事件（Error Events）
    ├── 崩溃事件
    ├── 性能事件
    └── 业务错误
```

### 2.2 详细事件定义

#### 2.2.1 用户事件

| 事件ID | 事件名称 | 触发时机 | 事件参数 |
|--------|----------|----------|----------|
| `app_launch` | 应用启动 | 用户打开APP | `source`(启动来源), `version` |
| `app_background` | 应用后台 | APP 进入后台 | `duration`(前台时长) |
| `onboarding_start` | 引导开始 | 用户开始新手引导 | `step`(当前步骤) |
| `onboarding_complete` | 引导完成 | 用户完成新手引导 | `duration`(耗时) |
| `onboarding_skip` | 引导跳过 | 用户跳过引导 | `skip_step`(跳过步骤) |
| `settings_change` | 设置变更 | 用户修改设置 | `key`(设置项), `value`(新值) |

#### 2.2.2 键盘事件

| 事件ID | 事件名称 | 触发时机 | 事件参数 |
|--------|----------|----------|----------|
| `keyboard_show` | 键盘呼出 | 键盘成功展示 | `trigger`(触发方式), `from`(来源页面) |
| `keyboard_hide` | 键盘隐藏 | 键盘被隐藏 | `duration`(展示时长), `action`(用户操作) |
| `clipboard_read` | 剪贴板读取 | 自动/手动读取剪贴板 | `source`(来源), `content_type`(内容类型), `length`(长度) |
| `clipboard_empty` | 剪贴板空 | 剪贴板无内容 | - |
| `message_input` | 手动输入 | 用户手动输入消息 | `length`(字符数) |

**触发方式枚举**：
- `keyboard_show` - 键盘弹出（点击输入框）
- `shortcut` - 快捷键
- `notification` - 通知栏
- `settings` - 设置页进入

#### 2.2.3 人设事件

| 事件ID | 事件名称 | 触发时机 | 事件参数 |
|--------|----------|----------|----------|
| `persona_create` | 创建人设 | 用户新建人设卡片 | `name`(名称), `scene`(场景), `relation`(关系) |
| `persona_edit` | 编辑人设 | 用户修改人设 | `persona_id`, `edit_type`(修改类型) |
| `persona_delete` | 删除人设 | 用户删除人设 | `persona_id`, `reason`(可选删除原因) |
| `persona_select` | 选择人设 | 用户选中人设卡片 | `persona_id`, `from`(来源) |
| `persona_default_set` | 设为默认 | 用户设置默认人设 | `persona_id` |
| `tag_add` | 添加标签 | 用户添加性格标签 | `tag_id`, `level`(级别), `source`(预设/自定义) |
| `tag_remove` | 移除标签 | 用户移除标签 | `tag_id`, `persona_id` |
| `tag_level_change` | 级别调整 | 用户调整标签级别 | `tag_id`, `old_level`, `new_level` |

#### 2.2.4 AI 生成事件

| 事件ID | 事件名称 | 触发时机 | 事件参数 |
|--------|----------|----------|----------|
| `ai_generate_start` | 开始生成 | 用户点击生成按钮 | `persona_id`, `scene`, `has_context`(是否有上下文) |
| `ai_generate_success` | 生成成功 | AI 返回结果 | `duration`(耗时), `result_count`(结果条数), `scene` |
| `ai_generate_fail` | 生成失败 | AI 返回错误 | `error_code`, `error_msg`, `retry_count` |
| `ai_regenerate` | 重新生成 | 用户点击重新生成 | `regenerate_reason` |
| `reply_select` | 选择回复 | 用户选择一条回复 | `reply_index`(1/2/3), `persona_id`, `scene` |
| `reply_copy` | 复制回复 | 用户复制回复内容 | `reply_index`, `content_length` |
| `reply_edit` | 编辑回复 | 用户编辑回复后发送 | `edit_type`(微调/重写) |
| `reply_send` | 发送回复 | 用户发送回复 | `send_method`(直接发送/复制发送), `app_name`(目标APP) |

#### 2.2.5 上下文学习事件

| 事件ID | 事件名称 | 触发时机 | 事件参数 |
|--------|----------|----------|----------|
| `context_save` | 保存上下文 | 生成回复后自动保存 | `batch_id`, `message_count` |
| `context_batch_end` | 批次结束 | 对话批次结束 | `batch_id`, `duration`, `message_count` |
| `ai_analysis_trigger` | 分析触发 | AI 开始分析批次 | `batch_id`, `trigger_type`(定时/手动) |
| `ai_analysis_complete` | 分析完成 | AI 完成性格分析 | `batch_id`, `duration`, `insight_count` |
| `weight_suggestion_show` | 权重建议展示 | 向用户展示权重调整建议 | `suggestions`(建议条数) |
| `weight_suggestion_apply` | 应用建议 | 用户应用权重调整建议 | `applied_count`, `dismissed_count` |
| `weight_manual_change` | 手动调整 | 用户手动调整权重 | `tag_id`, `old_weight`, `new_weight`, `source` |

#### 2.2.6 意图识别事件

| 事件ID | 事件名称 | 触发时机 | 事件参数 |
|--------|----------|----------|----------|
| `intent_detected` | 意图识别 | 识别到对方意图 | `intent_type`, `confidence`, `keywords` |
| `intent_feedback_show` | 反馈展示 | 展示意图识别结果 | `intent_type`, `content` |
| `intent_feedback_action` | 反馈操作 | 用户对反馈进行操作 | `intent_type`, `action`(查看/忽略/设置) |

**意图类型枚举**：
- `price` - 价格意图
- `time` - 时间意图
- `confirm` - 确认意图
- `emotion` - 情绪意图
- `promise` - 承诺意图

#### 2.2.7 业务事件

| 事件ID | 事件名称 | 触发时机 | 事件参数 |
|--------|----------|----------|----------|
| `pay_start` | 支付开始 | 用户点击购买 | `product_id`, `price`, `currency` |
| `pay_success` | 支付成功 | 支付完成 | `product_id`, `order_id`, `amount` |
| `pay_fail` | 支付失败 | 支付取消/失败 | `product_id`, `fail_reason` |
| `trial_start` | 试用开始 | 用户开启试用 | `product_id`, `trial_days` |
| `trial_expire` | 试用过期 | 试用期结束 | `product_id`, `used_days` |
| `share_invite` | 分享邀请 | 用户分享产品 | `share_channel`, `content_type` |
| `feedback_submit` | 反馈提交 | 用户提交反馈 | `type`(bug/feature/complain), `content` |
| `rating_submit` | 评分提交 | 用户提交评分 | `score`(1-5), `comment`(可选) |

#### 2.2.8 错误事件

| 事件ID | 事件名称 | 触发时机 | 事件参数 |
|--------|----------|----------|----------|
| `app_crash` | 应用崩溃 | 应用发生崩溃 | `crash_type`, `stack_trace`(可选) |
| `keyboard_error` | 键盘错误 | 键盘功能异常 | `error_type`, `error_msg` |
| `ai_error` | AI 错误 | AI 生成异常 | `error_type`, `error_msg`, `scene` |
| `network_error` | 网络错误 | 网络请求失败 | `api`, `error_code`, `retry_count` |

## 4. 关键漏斗分析

### 4.1 核心转化漏斗

```
漏斗1：AI 生成全流程
═══════════════════════════════════════════════════════════

┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  键盘呼出   │ → │  内容输入   │ → │ 点击生成   │ → │  选择回复  │
│  100%      │    │  85%       │    │  70%       │    │  60%       │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
      ↓                                    ↓                   ↓
   10,000          8,500              7,000              6,000

转化率分析：
- 键盘 → 内容：85% （15%直接离开，可能剪贴板为空）
- 内容 → 生成：82% （18%放弃，可能是手动输入太麻烦）
- 生成 → 选择：86% （14%未选择，可能对结果不满意）
```

| 漏斗阶段 | 事件 | 转化率 | 优化方向 |
|----------|------|--------|----------|
| 键盘呼出 → 内容输入 | keyboard_show → message_input/clipboard_read | 85% | 优化剪贴板读取体验 |
| 内容输入 → 点击生成 | message_input → ai_generate_start | 82% | 降低输入门槛 |
| 点击生成 → 选择回复 | ai_generate_success → reply_select | 86% | 提升生成质量 |
| 选择 → 发送 | reply_select → reply_send | 75% | 优化发送流程 |

## 5. 用户路径分析

### 5.1 核心用户路径

**路径1：首次使用流程**

```
启动APP → 新手引导 → 首次呼出键盘 → 读取剪贴板/手动输入
    → 生成回复 → 选择回复 → 发送/复制 → 完成激活
```

**路径2：日常使用流程**

```
呼出键盘 → 读取剪贴板 → 选择人设卡片 → 生成回复
    → 选择/编辑 → 发送/复制
```

**路径3：人设配置流程**

```
进入人设管理 → 创建新人设 → 设置关系 → 选择场景
    → 添加标签 → 设置权重 → 保存人设 → 设为默认
```

### 5.2 关键路径指标

| 路径 | 目标行为 | 关键节点 | 时长要求 |
|------|----------|----------|----------|
| 首次激活 | 发送回复 | 引导→生成→发送 | < 5分钟 |
| 日常使用 | 发送回复 | 呼出→生成→发送 | < 30秒 |
| 人设创建 | 保存人设 | 进入→保存 | < 3分钟 |

### 5.3 用户分群

| 分群维度 | 分群定义 | 运营策略 |
|----------|----------|----------|
| **活跃用户** | 近7天使用≥5次 | 维护关系，推送新功能 |
| **沉默用户** | 近7天使用≤1次 | 推送召回消息，分析流失原因 |
| **高价值用户** | 付费或分享≥3次 | VIP福利，专属客服 |
| **人设爱好者** | 创建≥3个人设 | 推送人设包、进阶教程 |
| **AI 重度用户** | 每天生成≥20次 | 引导付费，优化体验 |
